# Issue 2.3: Contract deployment pipeline (per-environment scripts + Basescan verification + Base Sepolia testnet deploys)

> One-line scope: take the three deploy scripts already shipped by P5-1 (v0), 2.1 (v1), and 2.2 (entitlement registry) and turn them into a **per-environment one-command bootstrap** — dev (local anvil), staging (Base Sepolia), and mainnet (TBD) — with **Basescan verification** baked into the same invocation. Run the testnet deploys of `RevisionLogV1` + `EntitlementRegistry` to Base Sepolia as part of this issue (analogous to P5-4 → D-014). Record the two new addresses as D-017 + D-018 in `DECISIONS.md` + `contracts/deployments/base-sepolia.json`. Mainnet deploy is gated on D-011 (external audit) + Kelvin authorization per §1.5; this issue lays the script + runbook so when the audit clears, the mainnet deploy is one well-tested command away.
>
> **Status:** Plan-gate **signed off by Kelvin on 2026-05-13.** All five Q-a..Q-e load-bearing questions resolved (see "Resolved decisions" below). Locked decisions L1..L8 + resolved R-a..R-d together are the non-negotiables. If anything in the "Approach" body drifts from a row in either table, the row wins. **Important interpretation of L7 + R-c**: the "issue owns the deploy" means the issue OWNS the runbook + the post-deploy D-NNN recording; the **actual deploy execution is a Kelvin-driven operational step after the code merges** (mirrors P5-4 → D-014: the issue shipped + merged, then Kelvin ran the deploy, then the address was recorded as a follow-up docs commit). The builder agent does NOT have access to the `pangolin-dev` keystore passphrase or the `BASESCAN_API_KEY`; those are Kelvin's secrets.
>
> **Security-critical: NO** (per master plan §16.3 — this is operational tooling, not contract code; the contracts being deployed already cleared §16 audits). The §16 cycle still applies but the **adversarial-audit bar is lighter**: deploy scripts have less attack surface than contracts (they don't run on-chain), but they DO produce on-chain artefacts that are immutable — so the audit focuses on "will this script land the right bytecode at the right network with the right constructor args" rather than full Solidity threat-modelling.
>

## Goal

A polished, auditable, repeatable deploy pipeline for all three contracts (v0 / v1 / entitlement registry) across three environments (dev / staging-Sepolia / mainnet-template), with Basescan verification baked into the deploy command and runbook documentation that future contributors can follow. Concretely, MVP-2 issue 2.3 ships:

1. **`scripts/deploy-contracts.sh`** — the one-command bootstrap (~150 LOC bash).
2. **`contracts/deploy/.env.dev`**, **`.env.sepolia`**, **`.env.mainnet.template`** — per-environment config.
3. **`contracts/deployments/base-sepolia.json` updates** — new entries for V1 + EntitlementRegistry alongside existing V0 (D-014 / D-015).
4. **`docs/RELEASE-CONTRACTS.md`** (new) — runbook covering pre-deploy checklist, deploy command, post-deploy verification, abort/rollback plan, mainnet-deploy guardrails.
5. **CI updates** — extend the existing `contracts-build` job's dry-run steps to invoke the wrapper script (not just `forge script`); verify the wrapper handles `--env dev --unattended` cleanly against a forge in-memory chain.
6. **The actual Base Sepolia testnet deploys** of V1 + EntitlementRegistry per L7.
7. **D-017 + D-018 entries in `DECISIONS.md`** recording the deployed addresses (analogous to D-014 + D-015).

**Non-goals:** mainnet deploy (gated on D-011 audit per L6 + master plan §1.5 — Kelvin's call); payment-processor + funder-service deployment (issue 3.4); `pangolin-chain` adapter pointing at the new addresses (separate MVP-2 integration issue — likely 3.1 or a new 2.x); any Rust crate changes (zero touched in 2.3).

## Approach

### 1. `scripts/deploy-contracts.sh` (new; ~150 LOC bash)

```bash
#!/usr/bin/env bash
set -euo pipefail

# Pangolin contract deploy pipeline (MVP-2 issue 2.3).
# Usage:
#   scripts/deploy-contracts.sh --env <dev|sepolia|mainnet> \
#                                --contract <v0|v1|entitlement|all> \
#                                [--unattended] [--dry-run]

# Parse flags
ENV=""
CONTRACT="all"
UNATTENDED=0
DRY_RUN=0
while [[ $# -gt 0 ]]; do
  case "$1" in
    --env) ENV="$2"; shift 2 ;;
    --contract) CONTRACT="$2"; shift 2 ;;
    --unattended) UNATTENDED=1; shift ;;
    --dry-run) DRY_RUN=1; shift ;;
    *) echo "Unknown flag: $1" >&2; exit 1 ;;
  esac
done

# Validate env
case "$ENV" in
  dev|sepolia) ;;
  mainnet)
    if [[ "${PANGOLIN_MAINNET_AUDITED:-0}" != "1" ]]; then
      echo "ERROR: --env mainnet requires PANGOLIN_MAINNET_AUDITED=1" >&2
      echo "       Set this only after the D-011 external audit has cleared." >&2
      exit 1
    fi
    ;;
  *) echo "ERROR: --env must be one of {dev, sepolia, mainnet}" >&2; exit 1 ;;
esac

# Source env file
ENV_FILE="contracts/deploy/.env.$ENV"
[[ -f "$ENV_FILE" ]] || { echo "Missing $ENV_FILE" >&2; exit 1; }
set -a; source "$ENV_FILE"; set +a

# Chain-id sanity check
ACTUAL_CHAIN_ID=$(cast chain-id --rpc-url "$RPC_URL")
[[ "$ACTUAL_CHAIN_ID" == "$EXPECTED_CHAIN_ID" ]] || {
  echo "ERROR: chain id mismatch — expected $EXPECTED_CHAIN_ID, got $ACTUAL_CHAIN_ID" >&2
  exit 1
}

# Deployer balance check (skip on dry-run)
if [[ "$DRY_RUN" == "0" && "$ENV" != "dev" ]]; then
  DEPLOYER_ADDR=$(cast wallet address --account "$DEPLOYER_ACCOUNT")
  BAL_WEI=$(cast balance "$DEPLOYER_ADDR" --rpc-url "$RPC_URL")
  MIN_WEI="${MIN_DEPLOYER_BALANCE_WEI:-10000000000000000}"  # 0.01 ETH default
  [[ "$BAL_WEI" -ge "$MIN_WEI" ]] || {
    echo "ERROR: deployer balance $BAL_WEI < required $MIN_WEI" >&2
    exit 1
  }
fi

# Deploy command flags
FORGE_FLAGS=(--rpc-url "$RPC_URL")
if [[ "$DRY_RUN" == "0" ]]; then
  FORGE_FLAGS+=(--broadcast --account "$DEPLOYER_ACCOUNT")
  if [[ -n "${BASESCAN_API_KEY:-}" ]]; then
    FORGE_FLAGS+=(--verify --etherscan-api-key "$BASESCAN_API_KEY" --verifier-url "$VERIFIER_URL")
  fi
fi

# Deploy each contract
deploy_one() {
  local script="$1" name="$2"
  echo "==> deploying $name to $ENV"
  forge script "contracts/script/${script}.s.sol" \
    --sig "run()" --tc "$script" \
    "${FORGE_FLAGS[@]}" \
    2>&1 | tee "/tmp/pangolin-deploy-${name}-${ENV}.log"
  # Parse address, tx hash, block from the log (forge prints these on broadcast)
  # ... record into contracts/deployments/${ENV}.json under appropriate key ...
}

case "$CONTRACT" in
  v0) deploy_one "DeployRevisionLogV0" "RevisionLogV0" ;;
  v1) deploy_one "DeployRevisionLogV1" "RevisionLogV1" ;;
  entitlement)
    # Pass authority addresses via env
    deploy_one "DeployEntitlementRegistry" "EntitlementRegistry"
    ;;
  all)
    deploy_one "DeployRevisionLogV1" "RevisionLogV1"
    deploy_one "DeployEntitlementRegistry" "EntitlementRegistry"
    ;;
  *) echo "ERROR: --contract must be one of {v0, v1, entitlement, all}" >&2; exit 1 ;;
esac

echo "DONE"
```

This is a sketch. The builder writes the production version with proper error handling, log parsing, JSON injection (via `jq` — already a dev dep), and post-deploy smoke tests.

### 4. `docs/RELEASE-CONTRACTS.md` (new runbook)

Sections:
- **Pre-deploy checklist** — ensure `pangolin-dev` keystore exists, has Base Sepolia funds (≥0.01 ETH), foundry pin is current, source is committed at the target HEAD.
- **Dev environment** — `scripts/deploy-contracts.sh --env dev --contract all --unattended` runs against an `anvil` instance; useful for integration testing.
- **Sepolia (staging)** — `scripts/deploy-contracts.sh --env sepolia --contract all` (prompts for keystore passphrase).
- **Mainnet** — gated checklist: external-audit clear; mainnet keystore created (hardware-wallet preferred); authority keys provisioned by billing + funder ops; sufficient deployer balance; rehearsal on Sepolia first; `PANGOLIN_MAINNET_AUDITED=1 scripts/deploy-contracts.sh --env mainnet --contract <one-at-a-time>`.
- **Post-deploy verification** — smoke-test commands (`cast call <addr> MAX_KNOWN_SCHEMA_VERSION()`); Basescan source-verified check; `contracts/deployments/{env}.json` updated; D-NNN entry added to `DECISIONS.md`.
- **Abort / rollback** — contracts are immutable; rollback = deploy v2 + update consumers + leave v1 as historical record (mirrors D-015 redeploy proof).
- **Operational notes** — gas-price strategy (peak Base congestion can push deploy cost to 0.05 ETH; cap with `--gas-price` if needed); RPC reliability (the public Base Sepolia RPC is sometimes flaky; alternates documented).

After all of the above lands + the parent's gate + audit clear, the wrapper runs:


### 7. Threat / failure modes

- **Wrong network deployed-to.** Guarded by chain-id check in the wrapper (`cast chain-id` vs `EXPECTED_CHAIN_ID`).
- **Wrong constructor args.** EntitlementRegistry has 2 constructor args; the wrapper validates `PAYMENT_AUTHORITY` and `REDEMPTION_AUTHORITY` are non-empty + valid-looking addresses before invoking forge.
- **Insufficient deployer balance.** Guarded by `cast balance` pre-check.
- **Accidental mainnet deploy.** Guarded by `PANGOLIN_MAINNET_AUDITED=1` env-var check (L6).
- **Basescan verification fails.** Non-fatal; address is recorded; manual `forge verify-contract` documented in the runbook.
- **Compromised deployer key.** Keystore-protected (passphrase prompt; `pangolin-dev` requires unlock); mainnet uses a separate keystore (hardware-wallet ideal).
- **`base-sepolia.json` race / corruption.** The wrapper uses `jq` + a tempfile + atomic rename pattern for the JSON update.
